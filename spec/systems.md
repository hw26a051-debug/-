# ゲームシステム仕様書 (systems.md)

## 1. ターゲット選択システム
- **デフォルトターゲット**: 画面内に存在するブロックのうち、**出現した時間（生成順）が最も古いブロック**が自動的に入力ターゲットとして選択される。
- **プレイヤーによる任意選択**: プレイヤーは、カーソル操作（マウスクリックやタップ等）によって、任意のブロックをアクティブなタイピングターゲットとして手動で選択できる。
- **ターゲットの視覚表示**: 現在タイピングの対象（アクティブ）となっているブロックは、他のブロックと区別できるように視覚的に強調表示（枠線、カラー変化など）される。

## 2. タイピングとブロック消去
- **入力方法**: 画面上のターゲットブロックに表示されている「ひらがな」に対応するローマ字をキーボードで入力する。
- **ローマ字入力の許容**:
  - 複数パターンのローマ字入力（例：「し」に対して "si" / "shi"、「じ」に対して "zi" / "ji"、「ち」に対して "ti" / "chi"、**「ふぉ」に対して "fo" / "fuxo" / "huxo"** など）に対応し、プレイヤーがどの入力方法で行っても正しく判定される。
- **ブロックの消去**: ターゲットブロックの単語をすべて正しく入力し終えると、そのブロックは画面から消去される。

## 3. スコア・コンボシステム
- **生存スコア**: ゲームオーバーになるまで生き残った時間（生存時間）に応じて、継続的にスコアが加算される。
- **消去スコア**: ブロックを消去するごとに、単語の文字数（消去したブロックのセル数）およびコンボ数に応じたスコアが加算される。
- **コンボシステム**: 
  - 単語を正しく入力し消去すると「コンボ」が1増加する。
  - コンボ数に応じて、獲得スコアに倍率などのボーナスが適用される。
- **ペナルティとコンボの途切れ条件**:
  - **タイプミス（打ち間違い）時のみ、コンボが0にリセット**される。
  - タイプミス以外（時間経過など）ではコンボは途切れない。
  - 打ち間違いによる入力制限などの、コンボリセット以外のペナルティは発生しない。

## 4. ゲームオーバー条件
- **ゲームオーバーライン**: プレイエリアの上部に「ゲームオーバーライン」が設定される。
- **終了条件**: 画面内に積み上がったブロックのいずれかが、このラインに接触、またはラインを超えた時点でゲームオーバーとなる。

## 5. ゲームモード
- **エンドレスモード限定**: ゲームモードは「エンドレスモード」のみの一本化とする。制限時間による終了や、面クリアなどのステージ制は採用せず、ゲームオーバーになるまで純粋にハイスコアを競い続ける設計とする。

## 6. ハイスコア・セーブシステム
- **ハイスコア保存機能**: プレイヤーの獲得した歴代最高スコア（ハイスコア）をブラウザのローカルストレージ（LocalStorage等）に自動的に保存・蓄積する。
- **ハイスコア削除（リセット）ボタン**: プレイヤーが任意でこれまでのセーブデータを初期化できるよう、スタート画面やリザルト画面、または設定パネル等に「ハイスコア削除（クリア）」ボタンを設置する。誤操作防止のため、クリック時に確認のダイアログ、または「本当に削除しますか？」という二段階確認（確認ステップ）を設ける。

## 7. フィーバー時の演出制限（プレイヤーの集中保護）
- **視覚・聴覚演出の不採用**: コンボが一定以上継続した際（フィーバー時）に、画面全体が激しく点滅したり、パーティクルが過剰に飛び散ったり、BGMが急激に変化するなどの「派手な演出」は**一切行わない**。
- **理由**: プレイヤーがタイピングとパズルの処理に極限まで集中できるようにするため、視覚的なノイズ（妨害要素）を排除する。
- **表現形式**: スコア加算値の数字表示やコンボ数カウンターなど、状況確認に必要な最低限のテキストUI（文字・数字）の更新のみに留める。
