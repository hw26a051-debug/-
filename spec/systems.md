# ゲームシステム仕様書 (systems.md)

## 1. ターゲット選択システム
- **デフォルトターゲット**: 画面内に存在するブロックのうち、**出現した時間（生成順）が最も古いブロック**が自動的に入力ターゲットとして選択される。
- **プレイヤーによる任意選択**: プレイヤーは、カーソル操作（マウスクリックやタップ等）によって、任意のブロックをアクティブなタイピングターゲットとして手動で選択できる。
- **ターゲットの視覚表示**: 現在タイピングの対象（アクティブ）となっているブロックは、他のブロックと区別できるように視覚的に強調表示（枠線、カラー変化など）される。

## 2. タイピングとブロック消去
- **入力方法**: 画面上のターゲットブロックに表示されている「ひらがな」に対応するローマ字をキーボードで入力する。
- **ローマ字入力の許容**:
  - 複数パターンのローマ字入力（例：「し」に対して "si" / "shi"、「じ」に対して "zi" / "ji"、「ち」に対して "ti" / "chi" など）に対応し、プレイヤーがどの入力方法で行っても正しく判定される。
- **ブロックの消去**: ターゲットブロックの単語をすべて正しく入力し終えると、そのブロックは画面から消去される。

## 3. スコア・コンボシステム
- **生存スコア**: ゲームオーバーになるまで生き残った時間（生存時間）に応じて、継続的にスコアが加算される。
- **消去スコア**: ブロックを消去するごとに、単語の文字数（消去したブロックのセル数）およびコンボ数に応じたスコアが加算される。
- **コンボシステム**: 
  - 単語を正しく入力し消去すると「コンボ」が1増加する。
  - コンボ数に応じて、獲得スコアに倍率などのボーナスが適用される。
- **ペナルティとコンボの途切れ条件**:
  - **タイプミス（打ち間違い）時のみ、コンボが0にリセット**される。
  - タイプミス以外（時間経過など）ではコンボは途切れない。
  - 打ち間違いによる入力制限などの、コンボリセット以外のペナルティは発生しない。

## 4. ゲームオーバー条件
- **ゲームオーバーライン**: プレイエリアの上部に「ゲームオーバーライン」が設定される。
- **終了条件**: 画面内に積み上がったブロックのいずれかが、このラインに接触、またはラインを超えた時点でゲームオーバーとなる。
