# 未確定事項・質問事項の解決状況 (questions.md)

本ドキュメントに記載されていたすべての疑問事項は、ゲームデザイナー兼プログラマーの協議により以下の通り決定・解決されました。

## 1. ブロックの形状について 【解決済み】
- **形状ルール**: 完全に直線のみではなく、**直線型（棒状）をメイン**とし、屈折した形状については**「L字型」のみ**をランダムで出現させる。
- **文字数の制限**: 出現する単語の長さは**最小3文字〜最大7文字**とする。

## 2. ターゲット選択システムについて 【解決済み】
- **「一番古いブロック」の定義**: **「画面に出現した時間（生成順）が最も古いブロック」**をデフォルトの入力ターゲットとする。
- **ターゲットの切り替え**: 自動選択に加え、プレイヤーがマウスやタップ等の**カーソル操作により、任意のブロックをクリックして手動でターゲットに選択できる**。
- **タイプミス時のペナルティ**: **コンボ数が0にリセットされるのみ**とし、入力ロックなどの追加ペナルティは発生しない。

## 3. 入力方式について 【解決済み】
- **ローマ字入力の揺れ**: "si" / "shi" などの複数通りのローマ字入力パターンに完全対応し、プレイヤーのタイピング習慣に合わせる。「ふぉ」については `fo` / `fuxo` / `huxo` / `huo` の全パターンに対応する。

## 4. スコア・コンボについて 【解決済み】
- **コンボの途切れ条件**: **タイプミス（打ち間違い）をした瞬間のみ**コンボが0にリセットされる。時間経過等ではコンボは維持される。

## 5. ハイスコアセーブと表示制限について 【解決済み】
- **プレイ中のハイスコア表示**: 画面を極力シンプルに保ち、プレイヤーの余計な焦りを防ぐため、**ゲームプレイ中のハイスコアは非表示**とする（タイトル画面およびゲームオーバー画面のみに表示）。
- **ハイスコア削除（リセット）ボタンの配置**: 誤クリックを防止するため、**タイトル（スタート）画面のみ**に配置する。

## 6. 一時停止（ポーズ）機能 【解決済み】
- **ポーズ機能の実装**: エンドレスモードでプレイが長引くことを考慮し、PCの `Esc` キー押下、または画面上のボタン操作によってゲーム進行を一時停止・再開できるポーズ機能を搭載する。ポーズ中はタイピング入力や落下判定も一時停止する。

## 7. 追加語彙と表記統一 【解決済み】
- **追加単語**: 「おにぎり」「フォルダ」「じどうしゃ」「ナポリタン」を単語リストへ追加。
- **表記ルール**: プレイヤーがタイピング対象を直感的に識別しやすく混乱を防ぐため、カタカナで表記できる語彙（スマートフォン、フォルダなど）はカタカナ表記に統一する。

## 8. ゲームオーバー時の画面遷移 【解決済み】
- **タイトルに戻るボタンの追加**: ゲームオーバー画面において、「リトライ」ボタンの左隣に「タイトルに戻る」ボタンを追加し、タイトル画面へいつでも遷移できるようにする。
